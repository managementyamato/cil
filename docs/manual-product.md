## 1. はじめに

| 損益管理 | OK | OK | - |

## 2. ログイン方法

### 手順

1. ブラウザでシステムのURLにアクセス
2. 「Googleでログイン」ボタンをクリック
3. 会社のGoogleアカウント（@yamato-mgt.com）でログイン
4. ダッシュボードが表示されれば成功

### セッションについて

- **8時間**操作がないと自動ログアウト
- 不審なログインはメール通知あり

## 4. プロジェクト管理

案件（プロジェクト）を管理する画面です。P番号で管理します。

### 画面への行き方

サイドバーの「プロジェクト管理」をクリック

### 一覧画面

```
┌─────────────────────────────────────────────────────────────────────┐
│ プロジェクト管理                                    [+ 新規追加]   │
├─────────────────────────────────────────────────────────────────────┤
│ [検索: ________] [ステータス: 全て ▼]                              │
├─────────────────────────────────────────────────────────────────────┤
│ P番号 │ 現場名     │ 顧客名   │ステータス │ 営業担当 │ 発生日   │
│───────┼────────────┼──────────┼───────────┼──────────┼──────────│
│ P123  │ A社ビル    │ A社     │ 成約     │ 山田    │ 2026/01/15│
│ P124  │ B店舗     │ B社     │ 設置予定 │ 佐藤    │ 2026/01/20│
│ P125  │ C工場     │ C社     │ 完了     │ 田中    │ 2026/01/25│
└─────────────────────────────────────────────────────────────────────┘
```

### ステータスの流れ

```
案件発生 → 成約 → 製品手配中 → 設置予定 → 設置済 → 完了
```

| ステータス | 意味 |
|-----------|------|
| 案件発生 | 案件が発生した状態 |
| 成約 | 契約が決まった状態 |
| 製品手配中 | 製品を手配している状態 |
| 設置予定 | 設置日が決まった状態 |
| 設置済 | 設置が完了した状態 |
| 完了 | すべての作業が完了 |

### 新規プロジェクトを追加する

1. 「+ 新規追加」ボタンをクリック
2. 必要な情報を入力

#### 入力項目

**基本情報**
| 項目 | 必須 | 説明 |
|------|:----:|------|
| P番号 | - | 自動採番（手動入力も可） |
| 発生日 | - | 案件が発生した日 |
| 取引形態 | - | 直販 / 代理店経由 など |

**担当・取引先**
| 項目 | 説明 |
|------|------|
| 営業担当 | 担当営業の名前 |
| 顧客名 | エンドユーザー名 |
| 販売店 | 代理店名 |
| 元請 | 元請業者名 |

**現場情報**
| 項目 | 説明 |
|------|------|
| 現場名 | 設置場所の名前 |
| 郵便番号 | 現場の郵便番号 |
| 都道府県 | 現場の都道府県 |
| 住所 | 現場の住所 |
| 納品先住所 | 製品の納品先（現場と異なる場合） |

**商品情報**
| 項目 | 説明 |
|------|------|
| 商品カテゴリ | 製品のカテゴリ |
| シリーズ | 製品シリーズ |
| 商品名 | 製品名 |
| 仕様 | 製品の仕様 |

**パートナー情報**
| 項目 | 説明 |
|------|------|
| 設置業者 | 設置を行う業者 |
| 撤去業者 | 撤去を行う業者 |

**日付情報**
| 項目 | 説明 |
|------|------|
| 成約日 | 契約日 |
| 設置予定日 | 設置予定日 |
| 設置完了日 | 設置完了日 |
| 出荷日 | 製品出荷日 |
| 保証終了日 | 保証期間の終了日 |

3. 「保存」をクリック

### プロジェクトを編集する

1. 編集したいプロジェクトの行をクリック
2. 編集モーダルが開く
3. 必要な項目を修正
4. 「保存」をクリック

### Google Chatスペース連携

プロジェクトごとにGoogle Chatスペースを作成・連携できます。

1. プロジェクトの編集画面を開く
2. 「Chatスペース」の項目を確認
3. 「新規作成」または「既存スペースを選択」
4. 保存すると自動的にスペースが作成/連携されます

## 5. トラブル対応

発生したトラブルを記録・管理する画面です。

### 画面への行き方

サイドバーの「トラブル対応」をクリック

### 一覧画面

```
┌─────────────────────────────────────────────────────────────────────┐
│ トラブル対応                                       [+ トラブル登録]│
├─────────────────────────────────────────────────────────────────────┤
│ [検索: ________] [ステータス: 全て ▼] [担当者: 全て ▼]            │
├─────────────────────────────────────────────────────────────────────┤
│ □ │ 日付 │ P番号 │ 内容       │ 対応者│ステータス│ 期限  │ 操作 │
│───┼──────┼───────┼────────────┼───────┼─────────┼───────┼──────│
│ □ │02/05 │ P123  │ 設置不良   │ 山田  │ 未対応  │ 02/10 │ 編集 │
│ □ │02/04 │ P456  │ 部品交換   │ 佐藤  │ 対応中  │ 02/08 │ 編集 │
│ □ │02/03 │ P789  │ クレーム   │ 田中  │ 完了    │ -     │ 編集 │
└─────────────────────────────────────────────────────────────────────┘
```

### ステータスの意味

| ステータス | 意味 | 色 |
|-----------|------|-----|
| 未対応 | 対応が始まっていない | 赤 |
| 対応中 | 対応作業中 | 黄 |
| 保留 | 一時的に対応を止めている | グレー |
| 完了 | 対応が終わった | 緑 |

### 新規トラブルを登録する

1. 「+ トラブル登録」ボタンをクリック
2. 必要な情報を入力：

| 項目 | 説明 |
|------|------|
| 日付 | トラブル発生日 |
| P番号 | 関連するプロジェクト番号 |
| トラブル内容 | 何が起きたか |
| 報告者 | 誰が報告したか |
| 対応者 | 誰が対応するか |
| 期限 | いつまでに対応するか |
| 会社名 | 関連する会社名 |
| 担当者名 | お客様担当者名 |

3. 「保存」をクリック

### ステータスを変更する

**方法1: 一覧から直接変更**
1. トラブルの「ステータス」セルをクリック
2. ドロップダウンから新しいステータスを選択

**方法2: 編集画面から変更**
1. 「編集」ボタンをクリック
2. ステータスを変更
3. 「保存」をクリック

### 担当者を変更する

1. トラブルの「対応者」セルをクリック
2. ドロップダウンから新しい担当者を選択

### 一括変更機能

複数のトラブルをまとめて変更できます。

1. 変更したいトラブルにチェックを入れる
2. 画面下部の一括操作パネルが表示される
3. 変更内容を選択：
   - 「担当者を変更」：担当者を一括変更
   - 「ステータスを変更」：ステータスを一括変更
4. 「実行」をクリック

```
┌─────────────────────────────────────────────────────────────────────┐
│ 選択中: 3件                                                        │
│                                                                     │
│ 担当者を変更: [担当者を選択 ▼]   ステータスを変更: [選択 ▼]       │
│                                                                     │
│                                          [キャンセル] [実行]       │
└─────────────────────────────────────────────────────────────────────┘
```

### 製品管理部でできること・できないこと

| 操作 | 可否 |
|------|------|
| 一覧を見る | OK |
| 詳細を見る | OK |
| トラブル登録 | OK |
| ステータス変更 | OK |
| 担当者変更 | OK |
| 内容の編集 | OK |
| 一括変更 | OK |
| 削除 | できません（管理部のみ） |

請求書とプロジェクトを紐付けることができます。

1. 請求書の「P番号」セルをクリック
2. プロジェクトを選択
3. 紐付けが保存される

## 9. アルコールチェック

従業員のアルコールチェック記録を管理する画面です。Google Chatと連携しています。

### 画面への行き方

サイドバーの「アルコールチェック」をクリック

### 画面の見方

```
┌─────────────────────────────────────────────────────────────────────┐
│ アルコールチェック                                                 │
├─────────────────────────────────────────────────────────────────────┤
│ 日付: [2026-02-06]  [◀ 前日] [翌日 ▶]           [Chatから同期]    │
├─────────────────────────────────────────────────────────────────────┤
│ ┌──────────┬──────────┬──────────┬──────────┬──────────┐          │
│ │ 従業員名 │ 提出状況 │ 結果     │ 時刻     │ 画像     │          │
│ ├──────────┼──────────┼──────────┼──────────┼──────────┤          │
│ │ 山田太郎 │ ✓ 提出済 │ 0.00     │ 08:15    │ [表示]   │          │
│ │ 佐藤花子 │ ✓ 提出済 │ 0.00     │ 08:30    │ [表示]   │          │
│ │ 田中一郎 │ ✗ 未提出 │ -        │ -        │ -        │          │
│ │ 鈴木次郎 │ 車不使用 │ -        │ -        │ -        │          │
│ └──────────┴──────────┴──────────┴──────────┴──────────┘          │
└─────────────────────────────────────────────────────────────────────┘
```

### 提出状況

| ステータス | 意味 |
|-----------|------|
| ✓ 提出済 | チェック画像を提出済み |
| ✗ 未提出 | まだ提出していない |
| 車不使用 | その日は車を使用しない |

### Google Chatから同期する

1. 「Chatから同期」ボタンをクリック
2. Google Chatに投稿された画像を取得
3. 従業員と自動的に紐付け

**紐付けの仕組み**:
- Google Chatの投稿者メールアドレスと従業員のメールアドレスを照合
- 自動的に正しい従業員に紐付けられる

### 「車不使用」を登録する

車を使わない日は「車不使用」として登録できます。

1. 従業員の行にある「車不使用」ボタンをクリック
2. その日は「未提出」としてカウントされなくなる

## 12. よくある質問（FAQ）

### Q1. トラブルを削除したいです

**A**: 製品管理部の権限では削除できません。管理部に依頼してください。

---

### Q2. MFからの同期がエラーになります

**A**: MF連携の認証が切れている可能性があります。管理部に連絡して、MF連携の再設定を依頼してください。

---

### Q3. アルコールチェックで従業員が紐付かない

**A**: 従業員マスタのメールアドレスとGoogle Chatのアカウントが一致している必要があります。
従業員マスタでメールアドレスを確認・修正してください。

---

### Q4. P番号を間違えて登録してしまいました

**A**: 編集画面でP番号以外の項目は修正できます。
P番号自体を変更する必要がある場合は、管理部に相談してください。

---

### Q5. 一括変更で間違えて変更してしまいました

**A**: 操作の取り消し機能があります。管理部に依頼してください。
（監査ログから変更履歴を確認し、元に戻すことができます）

---

### Q6. 新しい顧客や商品を登録したい

**A**: マスタ管理から登録できます。
1. サイドバーの「マスタ管理」をクリック
2. 該当するマスタ（顧客/商品/パートナー）を選択
3. 「+ 新規追加」ボタンをクリック
4. 情報を入力して保存

---

### Q7. 請求書をプロジェクトに紐付けたい

**A**: 損益管理画面で紐付けできます。
1. 損益管理を開く
2. 請求書の「P番号」セルをクリック
3. 紐付けたいプロジェクトを選択

---

### Q8. Google Chatスペースを案件に紐付けたい

**A**: プロジェクト管理画面で設定できます。
1. プロジェクトの編集画面を開く
2. 「Chatスペース」の項目で：
   - 「新規作成」：新しいスペースを自動作成
   - 「既存スペースを選択」：既存のスペースを紐付け
3. 保存
