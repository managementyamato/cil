### 2. 開発サーバーの二重起動問題

**症状**
ページネーションを実装したにもかかわらず50件以上が表示され続けた。

**原因**
PHPサーバープロセスが3つ同時に起動しており、うち1つが古い `C:\Claude\dev` ディレクトリから起動していた（`id="invoice-table"` が存在しない古いコードを配信）。

**修正内容**
- 全PHPプロセスを強制終了（PID: 5504, 21048, 38668）
- `scripts\start-dev-server.bat` から正しいディレクトリ `C:\Claude\master` で再起動

**対策**
開発サーバーは必ず `scripts\start-dev-server.bat` から起動し、他の方法では起動しない。

### 3. employees.php — 退職登録で HTTP 500 エラー

**症状**
従業員マスタで退職日を入力して「更新」ボタンを押すと `HTTP ERROR 500` が表示され、更新できなかった。

**原因**
`functions/audit-log.php` に以下のヘルパー関数が定義されておらず、`employees.php` から呼び出した時点で **Fatal Error: Call to undefined function** が発生し 500 エラーになっていた。

```
auditCreate()
auditUpdate()   ← employees.php の退職登録処理で呼び出し
auditDelete()
auditLogin()
auditLogout()
auditExport()
```

CLAUDE.md には使用例が記載されていたが、実装が抜け落ちていた。

**修正内容**
`functions/audit-log.php` の末尾に上記6つのヘルパー関数を追加。

```php
function auditCreate($entity, $id, $description, $newData = []) {
    writeAuditLog('create', $entity . '/' . $id, $description, ['new' => $newData]);
}

function auditUpdate($entity, $id, $description, $oldData = [], $newData = []) {
    writeAuditLog('update', $entity . '/' . $id, $description, ['old' => $oldData, 'new' => $newData]);
}

function auditDelete($entity, $id, $description, $deletedData = []) {
    writeAuditLog('delete', $entity . '/' . $id, $description, ['deleted' => $deletedData]);
}

function auditLogin($success, $email) {
    writeAuditLog($success ? 'login' : 'login_failed', 'auth/' . $email, ...);
}

function auditLogout() { ... }
function auditExport($entity, $format, $count) { ... }
```

**修正ファイル**
- `functions/audit-log.php`

**補足調査で判明したこと**
- 古い従業員データの `id` フィールドが `"emp_xxxxx"` 形式ではなく整数（`2`, `3`, `11` など）で保存されている
- PHP の厳密比較 `===` では文字列 `"2"` と整数 `2` が一致しないためIDマッチングが失敗するケースがある
- ただし `original_employee_code`（`YA-001` 等）による先行マッチングが正常に機能しているため実害はなかった
