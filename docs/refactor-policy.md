## 基本原則

### 1. 動作変更ゼロを最優先

- 既存の関数シグネチャ・戻り値・エラー挙動を一切変えない
- 新クラス・新ファイルの追加は OK、既存ファイルの内部実装を「整理」するのは原則 NG
- 「ついでに」のバグ修正・最適化は禁止。別 PR にする
- 公開 API（`getData()` / `saveData()` / `Database::*`）の振る舞いを変えない

### 2. ストラングラー・フィグ・パターン

- 既存ファイルを壊さない。新しい構造を横に作り、`require_once` / `include` で段階的に置き換える
- ロールバックが 1〜2行の削除で済む構成にする
- 例: `pages/sales-tools.php` は `pages/sales-tools/tabs/*.php` を `include` するだけのルーティング層に縮小する

### 3. データベース層の特例

- `config/database.php` の `saveEntity()` 内 `DB_SAVE_MODE` ロジックは **触らない**（CLAUDE.md 最重要 #0）
- 抽出して良いのは「読み取り専用ヘルパー」と「データ定義（配列）」のみ
- `saveEntityFullReplace` / `saveEntityUpsert` の中身は今回のリファクタ対象外

### 4. CLAUDE.md 最重要ルールの遵守

- `data.json` を直接触らない（最重要 #1）
- `DB_SAVE_MODE` のデフォルト値を変更しない（最重要 #0）
- 本番デプロイは営業時間外・即ロールバック可能体制で実施

## 着手前チェックリスト

各タスク着手前に以下を確認すること。1つでも NO があれば着手しない。

- [ ] 対応する E2E テストが存在する（または同時に追加する）
- [ ] 動作変更がないことを説明できる（diff レビュー可能な粒度になっている）
- [ ] ロールバック手順が明文化されている（git revert で1コミット戻すだけで済むか）
- [ ] 本番反映の時間帯が営業時間外か確認済み
- [ ] 関連ドキュメント（CLAUDE.md / docs/changelog.md）の更新が必要か判断済み

## 停止条件

リファクタリング作業中、以下の状態を検知した場合は即時停止し、原因分析を行う。

### 黄信号（要注意 / 一時停止判断）

| トリガー | 検知方法 | 対応 |
|---|---|---|
| 1000 行超ファイル数が**前週より増加** | `scripts/measure-metrics.php` の週次出力 | 増加要因を Slack 報告、要対応の場合 1 週間以内に巻き戻し |
| 直接ファイル操作違反数が**前週より増加** | 同上 | lint ルール（pre-commit）の漏れを確認、新規違反を弾く |
| E2E テストの**新規 fail が 1 件以上** | `npm test` の結果 | 該当 fail を 24h 以内に解消 or 該当変更を revert |
| 部下の作業時間が**見積もりの 1.3 倍を超過** | 週次レビューで部長確認 | スコープ縮小 or サポート増員を社長に上申 |

### 赤信号（即停止 / 全面ロールバック判断）

| トリガー | 検知方法 | 対応 |
|---|---|---|
| **本番障害発生** | エラーログ / ユーザー報告 | 即時 git revert、原因分析後に再開判断 |
| 1000 行超ファイル数が **2 週連続で増加** | 週次メトリクス | 部長判断で着手を一時停止、社長に状況報告 |
| `data.json` 不整合・破損 | `scripts/check-db-integrity.php` | 即時 `scripts/backup-data.php --restore` で復旧 |
| `Database::saveEntity` のエラーレート上昇 | サーバーログ | 即時該当変更を revert、UPSERT/full_replace 切替挙動を再確認 |
| カバレッジ率が **30% を下回る** | 週次メトリクス | 新規 E2E 追加を最優先に切替 |

### 黄信号と赤信号の違い

- **黄信号** = 様子見しつつ次の手を打つ余地がある状態。1週間以内の是正で OK。
- **赤信号** = 安全側に倒す判断。「先に止めて、後から原因を調べる」を徹底。

以下は Sprint 1 期間中、**変更しない**。スコープを広げない・狭めない。

- ✓ `config/database.php` のクラス分離（DBSaveModeManager / DualModeAdapter / JsonColumnHandler）

- ✗ `masters.php` / `finance.php` / `reports-hub.php` の分割（Sprint 2 へ）
- ✗ 既存 53 ファイルの直接ファイル操作違反の本格是正（Sprint 2 へ）

- ✗ Composer 導入・依存ライブラリの大幅変更
