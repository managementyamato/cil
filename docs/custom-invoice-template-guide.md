# 指定請求書テンプレート作成ガイド

## 概要

指定請求書はExcelテンプレートの「名前の定義」機能を使って、システム側がセル位置を自動認識します。取引先ごとに異なるレイアウトでも、正しく名前を付けておけばコード修正なしで動作します。

## 運用フロー

1. 取引先の指定フォーマット（.xlsx）を Drive の「指定請求書テンプレート」フォルダに配置
2. ファイル名は取引先名を含めること（例: `アクティオ.xlsx`、`西尾レントオール.xlsx`）
3. Excelで以下の「名前の定義」を設定（後述）
4. システムは MF請求書の `partner_name` と Drive内のファイル名を照合してテンプレを選ぶ

## 名前の定義 追加手順（Excel）

1. テンプレートxlsxを開く
2. 該当セル（または範囲）を選択
3. メニュー: 数式 → 名前の定義 → 新規作成
4. 「名前」欄に下表の名前を入力 → OK
5. 全ての名前を設定したら保存

## 設定する名前一覧

### 必須

| 名前 | 用途 | セル指定例 |
|---|---|---|
| `branch_name` | 営業所名（納入部門名）を入れるセル | 単セル `B5` または複数セル（1文字ずつ） |
| `items_table` | 明細表（ヘッダ行＋データ行の全体範囲） | 複数セル `B14:I26` など |

### 請求日（以下のいずれか）

パターン1: 単セルに通常の日付値を入れる
| 名前 | 用途 |
|---|---|
| `billing_date` | 請求日（単セル） |

パターン2: 年月日を分けて数字で入れる
| 名前 | 用途 |
|---|---|
| `billing_date_year` | 西暦4桁（4セル）|
| `billing_date_month` | 月2桁（2セル）|
| `billing_date_day` | 日2桁（2セル）|

### オプション

| 名前 | 用途 | セル指定 |
|---|---|---|
| `partner_code` | 取引先コード | 単セル or 複数セル |
| `registration_code` | 適格請求書登録番号（T+13桁）| 単セル or 複数セル |

## items_table の仕様

**ヘッダ行（範囲の1行目）**: 以下の文字列を含む列を自動認識します。

| ヘッダ文字列 | 対応フィールド | データ型 |
|---|---|---|
| 納入日 / 納品日 | delivery_date | 日付 |
| 品名 / 品目 | name | 文字列 |
| 軽減税率 | reduced_tax | ※（8%対象のみ） |
| 数量 | quantity | 数値 |
| 単価 | unit_price | 数値 |
| 金額 | amount | 数値 |
| 備考 | note | 文字列 |
| 注文No / 注文番号 | order_no | 文字列 |

**データ行**: ヘッダの次の行から最大行数までシステムが自動で明細を書き込みます。

**注意**:
- ヘッダ文字列はセル結合されていてもOK（結合セルの値を見る）
- 明細の行数が範囲を超える場合はエラー

## `_branches` シート（営業所マスタ、オプション）

同一テンプレで複数営業所に対応する場合、隠しシート名 `_branches` に以下を記載：

| A列（営業所名） | B列（取引先コード） |
|---|---|
| 宮崎営業所 | 1668202000 |
| 熊本営業所 | 1668202000 |
| 福岡営業所 | 1668202000 |
| ... | ... |

- `_branches` シートが存在する場合、指定請求書作成画面で営業所プルダウンに表示される
- MF請求書のタグ（「宮崎」等）と自動照合し、該当する営業所を初期選択
- 営業所選択時、B列の取引先コードが `partner_code` セルに自動セットされる
- `_branches` シートが無い場合、営業所欄は表示されない（営業所の概念がない取引先用）

## セル範囲の書き込みルール

システムは「名前の定義」の範囲（セル数）を見て書き込み方式を決定します。

**単セルの場合**: 値をそのまま書き込み
- 例: `branch_name` が `B5` 単セル → `B5` に「宮崎営業所」と書く

**複数セルの場合**: 1文字ずつ分割書き込み
- 例: `partner_code` が `BG11:BP11`（10セル）→ `1,6,6,8,2,0,2,0,0,0` と各セルに1桁ずつ
- 例: `branch_name` が 5セル範囲 → 「宮崎営業所」の各文字を順に

**日付の分割書き込み** (`billing_date_year` 等):
- 例: `billing_date_year` が4セル → `2026` の各桁を左から順に
- 範囲が文字数より少ない場合は切り捨て、多い場合は右詰め0パディング

**非連続範囲**: Excel の名前定義では「範囲A, 範囲B」のように複数範囲を指定可能ですが、シンプルにするため**連続範囲のみ**対応します。日付の 年月日ラベルを避けたい場合は `billing_date_year` / `_month` / `_day` を個別に名前定義してください。

## よくある質問

**Q. テンプレに既存データ（サンプル記入）がある場合、削除しないとダメ？**
A. システム書き込みでセル内容は上書きされるので問題ありません。ただし明細の `items_table` 範囲にはサンプル行を残さないこと（マージされたり複数取引先でバグ発生の原因）。

**Q. 画像・印鑑・罫線は残せる？**
A. 残せます。システムはセル値のみ書き換えるため、画像・罫線・フォント・セル結合・条件付き書式はすべて保持されます。

**Q. 数式は？**
A. 残せます。例えば「金額=数量×単価」「合計=SUM(金額列)」などは自動計算されます。ただし SUM 等の範囲が明細の全行をカバーしていることを確認してください。

**Q. 複数シートのxlsxでもOK？**
A. OK。システムは最初のシート（または `原本` という名前があればそれ）を使用します。不要なシートは事前削除推奨。

**Q. テンプレを修正したい**
A. Drive上のxlsxを編集・上書き保存するだけ。再デプロイ不要。
