状態別表示の差分(4パターン)

  • 通常: ランクバッジ色付 + 主担当AM + CCチップ表示
  • ランク未設定: 灰[未]バッジ + 黄帯「ランク未設定。管理部へ申請」
  • AM未割当: AM欄に赤字「未割当」+ CC警告下に「AM割当後にメール送信可」追加
  • API失敗: 全面グレーアウト + 赤帯「マスターサーバーに接続できません」

価格の扱い(重要)

  • 本資料には価格を記載しません。単価は別途「価格表」で管理
  • 価格表は「製品ごと・ランクごと(S層/A層/B層...)」の形式で整理
  • 例: モニたろう90インチ → S層:○○円 / A層:○○円 / B層:○○円
  • 顧客詳細画面ではランクのみ表示、営業はランクを見て価格表で単価を確認

UI上の必須注記文言(法務指定・常時表示)

  • ランク欄横: 「ランクは社内基準。顧客への開示は禁止」
  • 履歴ページ冒頭: 「変更履歴は監査証跡として7年保存」
  • 顧客一覧/詳細: 「価格は別途価格表を参照してください」

CC運用ルール(常時表示の根拠)

  • 第1条 見積・提案メールは主担当AMおよびCC候補全員をCCに入れる
  • 第2条 顧客からの価格交渉は24時間以内に主担当AMへ共有
  • 第3条 例外顧客(ニッケン/松濤/芝公園など)はCC候補に部長を必ず含める
  • 第4条 ランク変更要望は秘書経由で月次レビューに上げる

本体UI統合の仕組み

  • 本体 style.css と css/components.css を <link> で読込
  • CSS変数(--primary, --gray-*, --danger, --warning, --purple)使用
  • テーマ切替7色に追従(社長判断でテーマも併せて確定可能)
  • フォーム入力は .form-input、ボタンは本体 .btn/.role-badge 系規約準拠
